✅ GraphQL vs Restful API

* ** GraphQL 탄생배경: Restful API로는 `다양한 기종에 맞는 API`들을 `일일이 구현하는 것이 힘들어서`
* `GraphQL` 장점(+)
  * `하나의 End Point`(대부분)
  * `요청의 횟수`를 줄일 수 있다.
  * `요청의 크기`를 줄일 수 있다.
* `GraphQL` 단점(-)
  * `File` 등 Text 만으로 하기 힘든 내용들을 처리하기 복잡하다.
  * `재귀적인 쿼리가 불가능`하다.

* API를 구성할때 둘 중 하나의 방식만 가지고 갈 필요는 없으나, 범위를 잘 나누지 않는다면, 하나만 사용하는 것이 맞다.
